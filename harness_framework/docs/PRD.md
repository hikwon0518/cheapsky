# PRD: Cheapsky

> **인천 출발 아시아·미국 주요 20개 노선 항공권을, 사람이 큐레이션한 핫딜과 실시간 시세를 교차해서 "지금 저점인지"를 한 화면에서 증명하는 Discovery 대시보드.**

---

## 한 줄 요약
커뮤니티 관측 + 수동 시드 baseline 에 뽐뿌·(Stretch) 루리웹·플레이윙즈 큐레이션 딜을 교차하여, 가성비 탐험가에게 "오늘 인천에서 어디로 싸게 갈 수 있는지"를 세 가지 증거(분위수 수치·사회적 반응·시장 평균 대비)로 보여준다.

## 배경
- 한국 커뮤니티(뽐뿌·루리웹·클리앙)는 항공권 핫딜의 **2~3차 전파**라 지연·범위 편향이 심함
- 전문 블로그(플레이윙즈)가 더 빠른 1차 전파자
- 무료 사용 가능한 외부 시세 API 가 부재 (Amadeus for Developers 포털 가입 중단, Travelpayouts 등은 제휴 요구) — **ADR-022 Deprecated** 반영
- → **2-레이어 소스(블로그 큐레이션 + 커뮤니티) + Baseline(관측+수동 시드)** 구조로 재편. 시간이 갈수록 관측이 누적돼 baseline 정확도 자동 상승

## 포지셔닝
- ❌ 전세계 모든 노선 감시
- ❌ 특정 노선 가격 알림
- ✅ **인천 출발 아시아·미국 20개 노선**에 한정해, 오늘의 저점 딜을 **세 갈래 증거로 설명**하는 탐색 도구

## 목표
1. **주입** (3초): 히어로에 "오늘의 저점 딜 TOP 3"를 즉시 표시
2. **증명** (30초): 각 딜이 왜 저점인지 **수치·스파크라인·한 줄 맥락** 세 갈래로 설명
3. **범위** (탐색): 20개 노선의 오늘 시세를 하단 히트맵으로 한눈에
4. **신뢰**: FSC / LCC 분위수를 분리해서 배지 과잉 판정 방지. 클릭 한 번으로 계산 근거 공개

## 비목표

- 유럽·오세아니아·중동·남미 노선 (v2)
- 김포·제주 출발, 국내선 (v2)
- 사용자 계정·알림·즐겨찾기 (공유 버튼만)
- 상용 OTA 직접 크롤링 (API·공식 소스만)
- 다국어

## 데이터 소스 (2-레이어 + Baseline)

### 🎯 Baseline — 관측 데이터 + 수동 시드 (ADR-011)
- **관측 데이터** (`price_observations`) — 뽐뿌 크롤 데이터가 누적되어 노선별 FSC/LCC 분위수를 자연 형성. 30건 이상 누적되면 관측 단독 운영
- **수동 시드** (`baseline_seed.json`) — 20개 노선 × FSC/LCC/mixed 엔트리. 관측 콜드 스타트(<10건)일 때 사용. 조사 방법은 `methodology.md`
- **우선순위**: 관측 ≥30건 (high) → 관측 10~29건 혼합 (medium) → 시드 FSC/LCC (medium) → 시드 mixed (low, 🔥 미부여) → 없음 (🔥 미부여)
- ADR-022 Deprecated 로 Amadeus 계층은 Core 에 없음. Stretch 3 (외부 조건부) 에서 부활 검토

### ② 큐레이션 딜 — 플레이윙즈 (조건부)
- 1인 운영 블로그, 정형적 제목 구조
- **진행 전 필수 절차** (ADR-025):
  1. RSS 피드 탐색 → 있으면 RSS 소비 (크롤링 아님)
  2. 없으면 운영자 메일 통보 (범위·기간·트래픽 환원 명시)
  3. 10 영업일 무응답 → 평가 기간 한정 크롤링, 평가 종료 후 중단
  4. 거절 응답 → 영구 제외

### ③ 사회적 증거 — 뽐뿌 해외여행 + 루리웹 핫딜
- 반응(조회수·댓글)이 "사람이 관심 있는 딜" 신호
- Core는 **뽐뿌 1곳만**, 루리웹은 Stretch

## 🔥 저점 판정 — FSC / LCC 분리 (ADR-024)

기존 "90일 평균 대비 30% 할인" 단일 기준은 LCC 특가가 FSC 기준에 찍혀 과대 판정되는 문제. 해결:

```
딜 도착 → 운항사 파싱 (제목의 항공사명 → `lib/airlines.ts` 사전 조회)
  ↓
carrier_class 판정: FSC / LCC / mixed (미상)
  ↓
해당 클래스의 baseline 분위수 사용 (ADR-011 우선순위)
  - 관측 ≥30건 → 관측 단독 (confidence: high)
  - 관측 10~29건 → 관측·시드 혼합 0.6:0.4 (medium)
  - 관측 <10건 → 시드 FSC/LCC (medium) / 시드 mixed (low, 🔥 미부여)
  ↓
p10 이하 = 🔥 저점
```

**UI 표시**:
- 배지: `🔥 저점 -52%` (할인율, 직관적)
- 팝오버: `하위 8% (p8) · LCC 분위수 · 시장 기준 (관측 42건)` 또는 `… · 기준 (수동 조사)`

## 핵심 기능

### 1. 히어로 "오늘의 레이더" — 딜 전용
- 상단 고정: `오늘 찾은 저점 딜 <N>개 · 최대 할인율 <X>%`
- 로고 옆 서브라벨: `인천 출발 아시아·미국 저점 레이더`
- **TOP 3 딜 카드** (② + ③ 레이어에서만. 시세 카드(Stretch)는 히어로에 등장 X)
- 딜 0건: 히어로 섹션 렌더 생략 → `최근 7일 TOP 3` 폴백, 그것도 0건이면 히어로 숨김
- **모바일**: TOP 1만 크게, 2·3은 간소화

### 2. Community Picks 섹션 (Stretch)
- 뽐뿌·루리웹·플레이윙즈 기준 "반응 많은" 딜
- 카드에 `📍 뽐뿌 당일 조회 TOP` 같은 이진 라벨 (절대 조회수 숫자 표시 안 함 — 사이트간 스케일 상이)
- 히어로와 중복 카드는 `sources` 배열에 `playwings + ppomppu` 병합되어 "2곳 보도" 라벨

### 3. 오늘의 노선 시세 히트맵 (Stretch 2, 관측+시드 한정)
- 20개 노선 `ICN → {DEST}` baseline 기반 표시 (관측 누적 노선은 p50, 관측 부족 노선은 시드값)
- **데스크톱**: 5×4 그리드, 각 셀 색은 분위수 구간 (초록 p10↓ / 중립 p10~50 / 회색 p50+)
- **모바일**: 기본 접힘 상태. `노선 20곳 오늘 시세 보기 ∨` 버튼, 펼치면 리스트 형태 (좌측 분위수 점)
- 셀 클릭 → **스카이스캐너 검색 URL**로 이동 (예약은 외부, ToS 안전)
- 셀 카드에 `참고 시세` 라벨 명시 (딜 카드와 시각 구분)

### 4. 카드 유형 분리 (ADR-027)
| 유형 | 데이터 출처 | 클릭 시 | 라벨 | 배치 |
|------|-------------|---------|------|------|
| **딜 카드** | 플레이윙즈 / 뽐뿌 / 루리웹 | 원문 새 탭 | 출처 태그 | 히어로, 일반 리스트, Community Picks |
| **시세 카드** | `price_observations` + `baseline_seed.json` | 스카이스캐너 검색 URL | `참고 시세` | 시세 히트맵 섹션 전용 (Stretch 2) |

### 7. 카드 한 줄 맥락 (규칙 기반 + Stretch LLM)
- **Core (규칙 기반 폴백, 항상 표시)**: 데이터로부터 자동 생성한 정적 한 줄. 예: *"시장 평균 대비 -52% · 하위 p7 · LCC 분위수 기준"*
- **Stretch (LLM 큐레이션, 커버 가능 카드만)**: Claude Haiku 4.5가 자연스러운 한 줄로 재작성. 60자 이내, 사실 기반. 예: *"시장 평균 대비 52% 저렴. 지난 30일 이 노선 최저가."*
- LLM 생성 실패·규칙 위반 시 **규칙 폴백으로 자동 대체** — 카드에 한 줄이 빈 상태는 없음
- "Amadeus" 같은 API 명칭 노출 금지 (ADR-012)

### 8. 필터 5종 (ADR-020)
- 국가/대륙 (JP / SEA / 중화권 / 미국 / ALL)
- 최대 가격 (10/20/30/50/100만/무제한)
- 출발 월 (현재+6개월)
- 최소 할인율 (30/40/50%)
- 신선도 (24h / 7d / 30d)
- URL: `?region=JP&maxPrice=300000&month=2026-05&minDiscount=30&since=24h&t=<token>`

### 9. 실효성 검증 (Core: HEAD만, Stretch: GET+가격 패턴)
- 24시간+ 딜 3시간 주기 검증
- 상태: `active` / `snapshot` / `price_changed` / `unchecked`
- 3회 연속 snapshot → 조기 만료

### 10. 중복 제거 (ADR-009)
- `sha1(origin | dest | floor(price/1000)×1000 | YYYY-MM | carrier_class)`
- carrier_class 추가된 이유: 같은 노선·가격대라도 FSC/LCC는 별개 딜

### 11. 노선 빈도 마이크로 지표 (Stretch)
카드 하단 한 줄: `이 노선 30일 3번째 등장`

### 12. 공유 버튼
- Web Share API / 링크 복사
- URL에 필터 쿼리 + share token 포함

### 13. 신선도·크롤러 헬스
- 카드별 경과 시간 (KST)
- 푸터 헬스 점: 뽐뿌 / (Stretch 1) 루리웹 / 플레이윙즈 각각

### 14. 패닉 모드 (`SHOW_CACHED_ONLY=true`)
- env 한 줄로 모든 외부 호출 스킵
- DB 캐시만으로 UI 렌더
- 상단 `🔒 캐시 모드` 배너
- 시연 당일 외부 장애 대비

### 15. 아카이브 페이지 (Stretch)
- 매일 00:05 KST TOP 5 스냅샷 → `/archive/YYYY-MM-DD`
- 아카이브에 담긴 딜이 조기 만료(`verification_fail_count >= 3`)되거나 `snapshot` 상태여도 **아카이브 페이지에는 표시** (그날의 기록이므로). 카드에 `당시 가격` 라벨 병기

### 실패 경로
| 경로 | UX |
|------|-----|
| 크롤러 전체 2h+ 실패 | 히어로 숨김, 상단 `최근 수집 지연` 배너 |
| 관측 데이터 부족 (노선별 <10건) | 해당 노선은 시드 baseline 기반으로 `confidence: medium` 판정 |

| LLM 큐레이션 실패 | 규칙 기반 정적 한 줄로 자동 폴백 — 빈 카드 없음 |

| 아카이브 딜이 만료·snapshot | 아카이브 페이지엔 여전히 표시, `당시 가격` 라벨 |

| Share Token 틀림 | Basic Auth 프롬프트로 폴백 |
