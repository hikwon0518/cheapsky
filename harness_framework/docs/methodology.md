# Baseline 방법론

> 이 문서의 한 줄 요지: 🔥 저점 배지의 판정 기준이 되는 baseline 은 **커뮤니티 관측(`price_observations`)을 1차 소스**로 쓰고, 초기 관측이 빈약한 기간에는 수동 조사 시드(`data/baseline_seed.json`, 20 노선 × FSC/LCC/mixed)를 폴백으로 사용한다. **ADR-011 재작성(2026-04-18) + ADR-022 Rejected(2026-04-19, 외부 시세 API 영구 제외)** 반영.

---

## Baseline 우선순위 (요약, ADR-011)

| # | 소스 | 역할 | 신뢰도 |
|:-:|------|------|:------:|
| 1 | `price_observations` ≥ 30건 (carrier_class 매치) | **1차** — 관측 단독 | high |
| 2 | `price_observations` 10~29건 (carrier_class 매치) | 관측·시드 혼합 (`0.6·관측 + 0.4·시드`) | medium |
| 3 | `baseline_seed.json` (수동 조사, FSC/LCC 매치) | 관측 콜드 스타트 | medium |
| 4 | `baseline_seed.json` mixed 엔트리만 존재 | 문구 참조만 | low (🔥 미부여) |
| 5 | 없음 | — | 🔥 미부여 |

관측 ≥ 30건이 되면 시드 참조 중단. 관측 오염 감지 시 3순위(시드 단독)로 강제.

## 왜 수동 조사 시드가 필요한가 (외부 시세 API 영구 제외 후 역할 확대)
- **콜드 스타트**: Core 시작 직후 관측이 0~수 건인 기간의 🔥 판정 공백을 메움
- **재난 복구**: 커뮤니티 크롤 장애 또는 관측 오염 감지 시 시드 단독 강제 스위치의 근거 데이터
- **시연 안전망**: 시연 당일 외부 요인으로 관측 갱신이 늦어져도 baseline 기반 표시 유지
- 기존 "Amadeus 장애 폴백" 역할은 ADR-022 Rejected (2026-04-19) 로 영구 해소 — GDS 기반 API 자체가 Cheapsky 핵심 가치(핫딜 감지)와 구조적으로 분리되므로 시드+관측 단독 운영 확정

## 조사 절차

### 1) 데이터 소스
- **스카이스캐너 월별 최저가 뷰** — 노선별 향후 6개월 월별 최저가
- **네이버 항공권** — 월간 가격 그래프
- 두 소스의 **중간값** 사용. 한쪽만 조회 가능하면 단독

### 2) FSC / LCC 분리 조사
각 노선마다 세 가지 기준으로 분리 조사:
- FSC 시드: 스카이스캐너 "대한항공 + 아시아나 + 해당 노선 주요 외항 FSC" 필터 기준 평균
- LCC 시드: "진에어 + 제주항공 + 티웨이 + 에어부산 + 해당 노선 주요 외항 LCC" 필터 기준 평균
- mixed 시드: 전체 평균 (carrier_class 미상 딜에 사용)

`baseline_seed.json` 엔트리 구조:
```json
{
  "origin": "ICN",
  "destination": "KIX",
  "carrierClass": "lcc",
  "baselineKrw": 180000,
  "p10Krw": 99000,
  "p50Krw": 180000,
  "p90Krw": 280000,
  "confidence": "medium",
  "sampledAt": "2026-04-15",
  "source": "seed"
}
```

### 3) 샘플링 기간
- 조사일 기준 **과거 6개월 월별 최저가의 평균**
- 성수기(7·8·12·1월/설연휴/추석)는 제외 → 평시 기준 보수적 산정

### 4) 반올림
천 원 단위 반내림 (298,400 → 298,000).

### 5) 신뢰도
| 값 | 기준 | UI 반영 |
|----|------|---------|
| `high` | 시드는 정의상 medium 이하. high 는 관측 ≥ 30건 도달 시에만 baseline 전체로 승격 | — |
| `medium` | 시드 FSC/LCC 매치 (두 소스 모두 조회, 월별 ≥ 4개) | 🔥 부여 가능, 팝오버에 `medium` + `(수동 조사)` 표기 |
| `low` | 시드 mixed 엔트리만 존재 or 데이터 부족 | 🔥 **미부여**, `데이터 수집 중` 라벨 |

## 조사 이력
| 조사일 | 대상 | 조사자 | 원본 스프레드시트 | 비고 |
|--------|------|--------|-------------------|------|
| (최초 완료 시 기록) | 20개 × FSC/LCC/mixed ≈ 40~60 엔트리 | 프로젝트 소유자 | (링크) | FSC/LCC 필터 병행. 장거리 노선은 FSC+mixed 만 |

## 갱신 주기
- **분기 1회** 재조사. Core 초기에는 관측이 쌓이기 전이므로 시드가 실질적 기준선
- 노선별 `price_observations` ≥ 30건 누적되면 해당 노선은 관측 단독 운영, 시드 참조 중단
- 관측 10~29건 구간에서는 자동으로 혼합 (`0.6·관측 + 0.4·시드`)
- 스프레드시트 원본은 링크로만 첨부 (commit 안 함)

## 검증 방법 (평가자용)
1. 이 문서 대상·절차대로 1~2개 노선 재조사
2. `baseline_seed.json` 값과 비교
3. ±25% 이내면 합리적 범위 (시드는 초기 공백을 메우는 기준선이라 오차 허용 넓힘)

## 알려진 한계
- 수동 조사 조사자 편향
- 성수기 제외 기준 주관적
- 장거리(LAX·JFK·HNL) 직항 기준만 사용, 경유편 제외
- 별명 병합으로 특정 공항(HND·EWR 등) 가격 특성 희석
- **Core 초기 수 주는 시드가 baseline 대다수를 차지** → 시드 정확도가 UI confidence 에 직결

## 자동화 대체 여부
- 분기 1회 수동이 현실적. 자동화 우선순위 낮음
- **Stretch 3 (시세 API 재진입, ADR-026)** 성립 시 해당 API 를 월 1회 대량 호출로 시드 자동 재생성 가능 → 수동 조사 축소·폐지 검토 (BACKLOG)
